# Hermetic Server: demo-checkout

## Overview

This is the hermetic server configuration for `demo-checkout`. A hermetic server is an isolated, deterministic environment that mimics production but operates without external dependencies.

## Quick Start

```bash
# Build hermetic server
mvn clean package -Pharmetic

# Run with Docker Compose
docker-compose -f docker-compose.hermetic.yml up
```

## API Endpoints

### Seed Entities
```bash
POST /hermetic/seedEntities
Content-Type: application/json

{
  "entities": [
    {
      "entityId": "user-1",
      "entityType": "User",
      "entityValue": {"name": "Test User", "email": "test@example.com"}
    }
  ]
}
```

### Reset Data
```bash
POST /hermetic/reset
```

### Get Sample Data
```bash
GET /hermetic/sample-data/{entityType}
```

## Configuration

The hermetic server uses:
- H2 in-memory database
- Disabled external service calls
- Request/response tracing enabled
